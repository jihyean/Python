2. 가비지 컬렉터( Garbage Collector, GC )
	유효하지 않은(불필요한, 사용하지 않는) 메모리
	C언어와 달리 JAVA는 개발자가 메모리를 직접 해제하지 않음
	JVM의 GC가 가비지를 알아서 정리함
	GC가 메모리 낭비를 방지하기 위해 주기적으로 검사하여 메모리를 청소함
3. 동기 vs 비동기
	동기 : 일반적인 프로그래밍에서의 흐름
	비동기 : 주로 웹 브라우저 화면에서 사용자가 버튼을 눌렀을때, 해당 이벤트 대한 응답을 말함
		이벤트를 처리하는 동안에도 웹 브라우저 화면은 계속 서비스가되고있음
		=> 팀 프로젝트에서 사용한 비동기를 예시로들면 +++ !!
